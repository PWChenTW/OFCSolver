# Task Delegation: Pineapple OFC Fantasy Land 策略分析器

## 委派對象：data-specialist

## 任務背景
【重要】開始任務前，請先閱讀並理解核心開發原則：
1. MVP 優先 - 從最簡單的解決方案開始
2. 漸進式開發 - 快速迭代，每次只加一個小功能  
3. 批判性思考 - 質疑需求，提供更好的替代方案
4. 實用主義 - YAGNI原則，避免過度設計

## 具體任務
為 Pineapple OFC 的 Fantasy Land 設計優化策略分析器。

### Fantasy Land 基本規則
- 進入條件：前墩 QQ+（比標準 OFC 的 A+ 容易）
- 優勢：一次看到 14 張牌，可完美規劃
- 目標：最大化進入 FL 的機會，並儘可能維持

### 需要分析的策略要點
1. 如何平衡追求 QQ+ 與避免犯規的風險
2. 在不同街道（Streets）的決策優先級
3. Fantasy Land 中的最優擺放策略
4. 計算進入 FL 的期望值（EV）

## 預期產出
請基於核心開發原則提供**最簡單可行的解決方案**：

1. **基本策略分析算法**（避免過度複雜的數學模型）
2. **關鍵決策點識別**（重點關注實用性）
3. **簡單的 EV 計算方法**（先求正確，後求精確）
4. **MVP版本的實現方案**（能跑起來就行）

## 約束條件
- 第一版不需要考慮所有邊緣情況
- 優先實現核心邏輯，細節優化留待後續
- 避免過度工程，專注解決主要問題
- 如果需求過於複雜，請先提出簡化建議

## 技術要求
- 使用現有的 OFC 基礎類別
- 與現有代碼風格保持一致
- 提供清晰的註解說明邏輯
- 確保代碼可讀性優於性能優化
